Simulation File Builder

To run a Starfish simulation, you need to create a XML file in which you describe the details of the simulation and then load it into the Simulation Runner. This is a tool that generates the simulation XML file.

Structure of a Simulation File

Files consist of a sequence of commands which can either declare parameters or execute code. For example, the <time> command can be used to declare how many iterations the simulation will last and how many seconds each iteration is, and the <starfish> command will run the simulation.


Generally, simulation files will follow this structure:

  1. Declarations
  2. Running the simulation
  3. Output

Part 1: Declarations

First, declare the parameters of the simulation. Generally, simulations will contain these commands:

  • The <time> command is used to state how many iterations the simulation will last and how many seconds will pass per iteration.
  • Use the <domain> command to choose which coordinate space the simulation uses, and nest <mesh> commands in it to create meshes.
  • The <material> command defines types of materials and their physical characteristics.
  • The <boundary> command creates geometric objects in the simulation. These objects can be used, for example, as physical barriers particles can interact with, or a region where particles will appear.
  • Use the <sources> command to declare how particles are to be injected into the simulation.
  • Use these material interaction commands to define what happens when materials interact:
    • <surface_hit> for interactions between materials and a surface boundary.
    • <dmsc>for interactions between two particles.
    • <mcc> for interactions between a particle and a fluid.
    • <chemistry> for interactions between two fluids.

These commands are parsed in order, so if you say a boundary X is made up of a material Y, the boundary command must come after the material command because the material has not been defined yet.

Part 2: Running the simulation

After setting everything up in part 1, use the <starfish> command and the simulation will run.

Part 3: Output

After the simulation has finished running, tell Starfish which field scalars, vectors, etc. to write to a file using the <output> command.
